## 1. ОБЩЕЕ ОПИСАНИЕ

---

### 1.1 Суть проекта

Веб-приложение для совместного просмотра видео двумя людьми в реальном времени с живым чатом. Без регистрации, без рекламы, без лишнего — только скорость, удобство и кайф от просмотра.

### 1.2 Ключевые принципы

```
1. СКОРОСТЬ — всё должно летать, ноль задержек
2. УДОБСТВО — интуитивно понятно с первой секунды
3. НАДЁЖНОСТЬ — ничего не теряется, всё сохраняется в БД
4. МИНИМАЛИЗМ — ни одного лишнего элемента на экране
```

### 1.3 Пользователи

Всегда ровно 2 человека в комнате. Никакой публики.

### 1.4 Доступ

По прямой ссылке. Без авторизации, без пароля.

### 1.5 Платформы

| Платформа                          | Приоритет     | Требования                 |
| ---------------------------------- | ------------- | -------------------------- |
| ПК (Chrome, Firefox, Edge)         | Основная      | Полный функционал, 60fps   |
| Мобильный браузер (Chrome, Safari) | Обязательная  | Полный адаптив             |
| Планшет                            | Автоматически | Адаптив между ПК и мобилой |

```sql
CREATE TABLE rooms (
    id              VARCHAR(10) PRIMARY KEY,        -- nanoid
    created_at      TIMESTAMP DEFAULT NOW(),
    last_activity   TIMESTAMP DEFAULT NOW(),

    -- текущее состояние видео (для переподключения)

    current_video_title  TEXT,

-- ============================================
-- СООБЩЕНИЯ ЧАТА
-- ============================================
CREATE TABLE messages (
    id          VARCHAR(21) PRIMARY KEY,             -- nanoid
    room_id     VARCHAR(10) NOT NULL REFERENCES rooms(id) ON DELETE CASCADE,
    user_name   VARCHAR(20) NOT NULL,
    text        TEXT NOT NULL,
    type        VARCHAR(10) DEFAULT 'user',          -- user | system
    created_at  TIMESTAMP DEFAULT NOW()
);

-- Индекс для быстрой загрузки чата комнаты
CREATE INDEX idx_messages_room_id ON messages(room_id);
CREATE INDEX idx_messages_created_at ON messages(room_id, created_at);


-- ============================================
-- РЕАКЦИИ НА СООБЩЕНИЯ
-- ============================================
CREATE TABLE reactions (
    id          SERIAL PRIMARY KEY,

    user_name   VARCHAR(20) NOT NULL,
    emoji       VARCHAR(10) NOT NULL,
    created_at  TIMESTAMP DEFAULT NOW(),

CREATE INDEX idx_reactions_message_id ON reactions(message_id);

-- ПЛЕЙЛИСТ
-- ============================================
```

### 3.3 Стратегия работы с БД (производительность)

```
ПРИНЦИП: БД для персистентности, память для скорости

┌─────────────────────────────────────────────────────┐
│                    СЕРВЕР (Node.js)                  │
│                                                     │
│  ┌─────────────────────────────────────────────┐    │
│  │         RAM (Map<roomId, RoomState>)         │    │
│  │                                             │    │
│  │  Хранит:                                    │    │
│  │  - Активные комнаты                         │    │
│  │  - Подключённые сокеты                      │    │
│  │  - Текущее время видео                      │    │
│  │  - Статус play/pause                        │    │
│  │  - Последние 50 сообщений чата (кеш)        │    │
│  │                                             │    │
│  │  Скорость: <1ms                             │    │
│  └──────────────┬──────────────────────────────┘    │
│                 │                                    │
│                 │ Async write-behind                 │
│                 │ (не блокирует WebSocket)           │
│                 ▼                                    │
│  ┌─────────────────────────────────────────────┐    │
│  │              Neon PostgreSQL                 │    │
│  │                                             │    │
│  │  Хранит:                                    │    │
│  │  - Все сообщения чата (полная история)      │    │
│  │  - Историю просмотров                       │    │
│  │  - Плейлисты                                │    │
│  │  - Состояние комнат (для переподключения)   │    │
│  │                                             │    │
│  │  Скорость: 5-50ms                           │    │
│  └─────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘

ЧТЕНИЕ:
- Сообщения чата → сначала из RAM кеша → если нет, из БД
- Состояние видео → всегда из RAM (мгновенно)
- История просмотров → из БД (редко запрашивается)

ЗАПИСЬ:
- Новое сообщение → мгновенно в RAM + async в БД
- Play/pause/seek → только RAM (в БД каждые 10 сек)
- Новое видео в плейлист → RAM + async в БД
- История просмотра → async в БД (при смене видео)
```

### 3.4 Батчинг записи в БД

```javascript
// Сообщения чата НЕ пишутся по одному
// Накапливаются в буфере и пишутся пачкой каждые 5 секунд

class MessageBuffer {
  buffer = [];

  add(message) {
    this.buffer.push(message);
  }

  async flush() {
    if (this.buffer.length === 0) return;
    const batch = this.buffer.splice(0);
    await db.insert(messages).values(batch); // один INSERT на всю пачку
  }

  startInterval() {
    setInterval(() => this.flush(), 5000); // каждые 5 сек
  }
}

// Состояние видео пишется в БД каждые 10 секунд
// (чтобы при перезагрузке сервера восстановить позицию)
```

### 4.1 Модуль «Комнаты»

---

#### 4.1.1 Создание комнаты

```
Пользователь заходит на главную →
  Вводит имя (2–20 символов, сохраняется в localStorage) →
    Нажимает «Создать комнату» →
      POST /api/rooms → создаётся запись в БД →
        Редирект на /room/{id} →
          WebSocket подключение →
            Видит кнопку «Скопировать ссылку»
```

#### 4.1.2 Подключение к комнате

```
Второй открывает ссылку /room/{id} →
  Если не ввёл имя → модалка «Введите имя» →
    WebSocket подключение →
      Сервер проверяет: комната существует? < 2 человек? →
        Успех: получает полное состояние (видео + чат из БД + плейлист) →
        Ошибка: «Комната не найдена» или «Комната заполнена»
```

#### 4.1.3 При подключении к существующей комнате клиент получает

```json
{
  "room": {
    "id": "x7k9p2m4",
    "users": ["Алексей"]
  },
  "video": {
    "url": "https://youtube.com/watch?v=...",
    "title": "Название",
    "currentTime": 125.4,
    "isPlaying": false,
    "speed": 1.0
  },
  "messages": [
    // последние 100 сообщений из БД
  ],
  "playlist": [
    // текущий плейлист из БД
  ]
}
```

#### 4.1.4 Отключение и переподключение

| Ситуация                  | Поведение                                           |
| ------------------------- | --------------------------------------------------- |
| Один закрыл вкладку       | Второй видит «{Имя} отключился», видео на паузе     |
| Интернет мигнул (<15 сек) | Socket.io автореконнект, незаметно для пользователя |
| Оба ушли                  | Комната живёт в БД 30 дней                          |
| Вернулись через день      | Полное восстановление: чат, плейлист, история       |
| Render перезапустился     | RAM очищается, но всё восстанавливается из Neon     |

#### 4.1.5 Восстановление после рестарта сервера

```
1. Сервер запускается
2. Не загружает ничего из БД (lazy loading)
3. Когда клиент подключается к /room/{id}:
   - Проверяет RAM → нет комнаты
   - Загружает из Neon: rooms + messages + playlist_items
   - Кеширует в RAM
   - Отдаёт клиенту
4. Всё работает как будто рестарта не было
```

### 4.2 Модуль «Видеоплеер»

---

#### 4.2.1 Поддерживаемые источники

| Источник | Приоритет | Метод                          | Примеры URL                                                       |
| -------- | --------- | ------------------------------ | ----------------------------------------------------------------- |
| YouTube  | 5/5       | ReactPlayer iframe API         | `youtube.com/watch?v=...` `youtu.be/...` `youtube.com/shorts/...` |
| VK Видео | 5/5       | ReactPlayer / кастомный iframe | `vk.com/video-123_456` `vk.com/clip-123_456`                      |
| Rutube   | 5/5       | ReactPlayer / кастомный iframe | `rutube.ru/video/...`                                             |

#### 4.2.2 Поле ввода URL

**Расположение:** над видеоплеером, всегда видно

```
┌─────────────────────────────────────────────────────────┐
│ [🔍] Вставьте ссылку YouTube, VK или Rutube   [▶] [+]  │
└─────────────────────────────────────────────────────────┘
      │                                          │    │
      │                                          │    └─ В плейлист
      │                                          └─ Воспроизвести сейчас
      └─ Авто-определение источника + иконка
```

**Автоопределение:**

- При вставке ссылки (paste) → мгновенный парсинг
- Иконка источника появляется: 🔴 YouTube, 🔵 VK, 🟢 Rutube
- Невалидная ссылка → красная обводка + «Неподдерживаемая ссылка»
- Ctrl+V автоматически вставляет и парсит (не надо кликать на поле)

#### 4.2.3 Управление воспроизведением

| Действие           | Кто может  | Синхронизируется | Задержка |
| ------------------ | ---------- | ---------------- | -------- |
| Play               | Оба        | ✅ Да            | <100ms   |
| Pause              | Оба        | ✅ Да            | <100ms   |
| Seek (перемотка)   | Оба        | ✅ Да            | <200ms   |
| Громкость          | Каждый сам | ❌ Нет           | —        |
| Полный экран       | Каждый сам | ❌ Нет           | —        |
| Скорость (0.5x–2x) | Оба        | ✅ Да            | <100ms   |
| Следующее видео    | Оба        | ✅ Да            | <200ms   |
| Предыдущее видео   | Оба        | ✅ Да            | <200ms   |

#### 4.2.4 Элементы управления плеером

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│                 ВИДЕО ОБЛАСТЬ                       │
│           (клик = play/pause)                       │
│           (двойной клик = fullscreen)               │
│                                                     │
├─────────────────────────────────────────────────────┤
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │  ← прогресс-бар
├─────────────────────────────────────────────────────┤
│ ⏮  ▶/⏸  ⏭   🔊━━━━━   01:23 / 02:45   1x   ⛶  │
└─────────────────────────────────────────────────────┘
     │    │    │      │         │            │    │
     │    │    │      │         │            │    └─ Полный экран
     │    │    │      │         │            └─ Скорость (клик → меню)
     │    │    │      │         └─ Время текущее / общее
     │    │    │      └─ Громкость (hover → слайдер)
     │    │    └─ Следующее видео в плейлисте
     │    └─ Play / Pause
     └─ Предыдущее видео
```

**Прогресс-бар:**

- При наведении → показывает время в tooltip
- Можно кликнуть для перемотки
- Можно зажать и тащить
- Буферизованная часть показывается полупрозрачным цветом

#### 4.2.5 Горячие клавиши (только на ПК)

| Клавиша  | Действие           |
| -------- | ------------------ |
| `Пробел` | Play / Pause       |
| `F`      | Полный экран       |
| `M`      | Mute / Unmute      |
| `←`      | Перемотка -10 сек  |
| `→`      | Перемотка +10 сек  |
| `↑`      | Громкость +10%     |
| `↓`      | Громкость -10%     |
| `N`      | Следующее видео    |
| `P`      | Предыдущее видео   |
| `>`      | Увеличить скорость |
| `<`      | Уменьшить скорость |

**Важно:** горячие клавиши НЕ работают когда фокус в поле ввода чата или URL

```
Пользователь нажал PLAY:
  Client → Server: video:play { time: 45.2 }
  Server → Оба Client: video:play { time: 45.2, user: "Алексей" }
  Оба клиента: seekTo(45.2) + play()
  Чат: системное сообщение "▶ Алексей продолжил воспроизведение"

Пользователь нажал PAUSE:
  Client → Server: video:pause { time: 45.2 }
  Server → Оба Client: video:pause { time: 45.2, user: "Алексей" }
  Оба клиента: seekTo(45.2) + pause()
  Чат: системное сообщение "⏸ Алексей поставил на паузу"

Пользователь ПЕРЕМОТАЛ:
  Client → Server: video:seek { time: 3661.0 }
  Server → Оба Client: video:seek { time: 3661.0, user: "Алексей" }
  Оба клиента: seekTo(3661.0)
  Чат: системное сообщение "⏩ Алексей перемотал на 1:01:01"


═══════════════════════════════════════════════
ФОНОВАЯ СИНХРОНИЗАЦИЯ (drift correction)
═══════════════════════════════════════════════

Каждые 3 секунды (когда видео playing):
  Client → Server: video:timeUpdate { time: currentTime }

Сервер хранит authoritative time (среднее обоих клиентов)

Если разница между клиентами:
  < 1.0 сек  →  ничего не делать (норма)
  1.0–3.0 сек  →  мягкая коррекция (playbackRate 1.05x на 5 сек)
  > 3.0 сек  →  жёсткий seek к authoritative time


═══════════════════════════════════════════════
БУФЕРИЗАЦИЯ
═══════════════════════════════════════════════

Клиент начал буферизовать:
  Client → Server: video:buffering { isBuffering: true }
  Server → Другой Client: video:peerBuffering { user: "Дмитрий" }
  Другой клиент: pause() + показать "Дмитрий буферизует..."

Буферизация завершилась:
  Client → Server: video:buffering { isBuffering: false }
  Server → Оба: video:play { time: authoritative_time }
  Оба: seekTo(time) + play()
```

### 4.3 Модуль «Чат»

---

#### 4.3.1 Общие требования

- Расположение: справа от видео (ПК), под видео (мобайл)
- Ширина на ПК: 35–40% экрана
- Сообщения хранятся в Neon PostgreSQL (полная история)
- При заходе в комнату загружаются последние 100 сообщений из БД
- Подгрузка старых сообщений при скролле вверх (lazy loading, по 50 штук)
- Автоскролл к новому сообщению
- Если проскроллил вверх → автоскролл выключается → кнопка «↓ Новые сообщения (3)» с счётчиком

#### 4.3.2 Структура сообщения пользователя

```
┌──────────────────────────────────────┐
│ [АЛ]  Алексей                 23:45 │
│                                      │
│ Смотри какой момент крутой 😂        │
│                                      │
│                         😂2  ❤️1    │
└──────────────────────────────────────┘
```

**Компоненты:**

- `[АЛ]` — аватар (кружок 32px, цвет из хеша имени, первые 2 буквы)
- `Алексей` — имя, жирным
- `23:45` — время, серым, справа
- Текст сообщения — обычный шрифт
- Реакции — маленькие бейджики под сообщением

#### 4.3.3 Системные сообщения

```
─────── Алексей присоединился ───────
─── ▶ Начато: Название видео [YT] ───
────── ⏸ Дмитрий: пауза ──────
──── ⏩ Алексей перемотал на 1:23:45 ────
─────── Дмитрий отключился ───────
```

Стиль: по центру, серый текст, мелкий шрифт, без аватарки

#### 4.3.4 Аватарки

```javascript
// Генерация цвета из имени
function avatarColor(name) {
  let hash = 0;
  for (let char of name) hash = char.charCodeAt(0) + ((hash << 5) - hash);
  const hue = Math.abs(hash % 360);
  return `hsl(${hue}, 65%, 55%)`;
}

// Инициалы
function initials(name) {
  return name.slice(0, 2).toUpperCase();
}

// Пример: "Алексей" → [АЛ] с hsl(287, 65%, 55%)
// Пример: "Дмитрий" → [ДМ] с hsl(134, 65%, 55%)
```

#### 4.3.5 Эмодзи

**Пикер:**

- Кнопка 😊 слева от поля ввода
- При клике → всплывающая панель с категориями
- Категории: Часто используемые, Смайлы, Жесты, Сердечки, Животные, Еда
- Поиск по эмодзи
- Библиотека: emoji-mart

**Нативные эмодзи:**

- На мобиле — стандартная клавиатура с эмодзи (не нужен отдельный пикер)
- Эмодзи рендерятся нативно (без спрайтов — быстрее)

#### 4.3.6 Реакции на сообщения

**Как поставить реакцию:**

- ПК: hover на сообщение → появляется полоска с 6 быстрыми реакциями
- Мобайл: long press (500ms) → появляется та же полоска

**Быстрые реакции:** 😂 ❤️ 👍 👎 😮 🔥

**Логика:**

- Клик на реакцию → toggle (поставить/убрать)
- Каждый может поставить только одну реакцию на сообщение (UNIQUE constraint в БД)
- Если нажал другую → предыдущая заменяется
- Реакция мгновенно показывается + async сохраняется в БД
- Под сообщением: бейджики с эмодзи и числом `😂 2` `❤️ 1`

#### 4.3.7 Поле ввода сообщения

```
┌────────────────────────────────────────────┐
│ 😊 │ Написать сообщение...            │ ➤ │
└────────────────────────────────────────────┘
```

- `😊` — открыть пикер эмодзи
- Текстовое поле — auto-resize до 4 строк максимум
- `➤` — кнопка отправки (активна только если есть текст)
- `Enter` — отправить
- `Shift + Enter` — перенос строки
- Максимум 500 символов
- Пустое сообщение не отправляется
- После отправки фокус остаётся в поле ввода

### 7.1 Подключение и комнаты

#### Клиент → Сервер

| Событие      | Payload                                | Описание         |
| ------------ | -------------------------------------- | ---------------- |
| `room:join`  | `{ roomId: string, userName: string }` | Войти в комнату  |
| `room:leave` | —                                      | Покинуть комнату |

#### Сервер → Клиент

| Событие           | Payload                                   | Описание                   |
| ----------------- | ----------------------------------------- | -------------------------- |
| `room:state`      | `{ room, video, messages[], playlist[] }` | Полное состояние при входе |
| `room:userJoined` | `{ userName: string }`                    | Второй участник зашёл      |
| `room:userLeft`   | `{ userName: string }`                    | Участник вышел             |
| `room:full`       | —                                         | Комната заполнена (2/2)    |
| `room:notFound`   | —                                         | Комната не существует      |
| `room:error`      | `{ message: string }`                     | Ошибка                     |

### 7.2 Видео

#### Клиент → Сервер

| Событие            | Payload                                           |

| `video:seek`       | `{ time: number }`                                |
| `video:change`     | `{ url: string, title?: string, source: string }` |
| `video:speed`      | `{ speed: number }`                               |

## 9. ОПТИМИЗАЦИЯ ПРОИЗВОДИТЕЛЬНОСТИ

---

### 9.1 Frontend

| Оптимизация             | Что делаем                                | Эффект                        |
| ----------------------- | ----------------------------------------- | ----------------------------- |
| Code splitting          | React.lazy для History, EmojiPicker       | Начальный бандл -40%          |
| Виртуализация чата      | @tanstack/react-virtual для MessageList   | 1000+ сообщений без тормозов  |
| Memo                    | React.memo на ChatMessage, PlaylistItem   | Нет лишних ре-рендеров        |
| Debounce seek           | Debounce 200ms на прогресс-бар            | Меньше WebSocket событий      |
| Throttle timeUpdate     | Каждые 3 сек вместо каждого кадра         | Меньше нагрузки на сокет      |
| Оптимизация изображений | Маленькие превью (120x68px)               | Быстрая загрузка плейлиста    |
| Preconnect              | `<link rel="preconnect">` к Neon, YouTube | Быстрее первый запрос         |
| Service Worker          | Кеширование статики                       | Мгновенная повторная загрузка |

### 9.2 Backend

| Оптимизация            | Что делаем                                | Эффект                       |
| ---------------------- | ----------------------------------------- | ---------------------------- |
| Fastify вместо Express | В 2x быстрее обработка HTTP               | Меньше latency               |
| RAM кеш комнат         | Map<roomId, state>                        | Состояние видео за <1ms      |
| Message batching       | Пишем в БД пачками по 5 сек               | В 10x меньше INSERT'ов       |
| Connection pooling     | Neon serverless driver с HTTP             | Нет холодных TCP-подключений |
| Сжатие Socket.io       | `perMessageDeflate: true`                 | Меньше трафика               |
| Lazy room loading      | Загружаем из БД только при первом connect | Быстрый старт сервера        |

### 9.3 Сеть

| Оптимизация      | Что делаем                                |
| ---------------- | ----------------------------------------- |
| Binary WebSocket | Для timeUpdate использовать binary frames |
| Gzip             | Fastify compress для REST ответов         |
| CDN для статики  | Render Static Site = CDN из коробки       |

## 10. ОБРАБОТКА ОШИБОК И EDGE CASES

---

### 10.1 Сетевые ошибки

| Ситуация                           | Поведение                                                        |
| ---------------------------------- | ---------------------------------------------------------------- |
| WebSocket отключился               | Автореконнект (1с, 3с, 5с, 10с). Toast: «Переподключение...»     |
| WebSocket не смог переподключиться | Toast: «Соединение потеряно». Кнопка «Обновить страницу»         |
| REST API не отвечает               | Toast: «Сервер недоступен». Retry через 5 сек                    |
| Render холодный старт              | Полноэкранный лоадер «Просыпаемся... ☕» с анимацией (до 30 сек) |
| Neon холодный старт                | Retry запросы к БД (3 попытки с exponential backoff)             |

### 10.2 Видео ошибки

| Ситуация                       | Поведение                                                             |
| ------------------------------ | --------------------------------------------------------------------- |
| Видео не загружается (404)     | Toast: «Видео недоступно». Предложить следующее из плейлиста          |
| Embed заблокирован (VK/Rutube) | Toast: «Не удалось встроить видео» + кнопка «Открыть в новой вкладке» |
| Невалидная ссылка              | Красная обводка поля + текст «Неподдерживаемая ссылка»                |
| Видео удалено с платформы      | Такое же как 404                                                      |

### 10.3 Чат ошибки

| Ситуация                 | Поведение                                                     |
| ------------------------ | ------------------------------------------------------------- |
| Сообщение не отправилось | Показать серым с иконкой ⚠️ + «Нажмите чтобы повторить»       |
| Превышен rate limit      | Toast: «Подождите немного»                                    |
| БД недоступна для записи | Сообщения работают через RAM, пишутся в БД при восстановлении |

### 10.4 Комната ошибки

| Ситуация                | Поведение                                                   |
| ----------------------- | ----------------------------------------------------------- |
| Комната не найдена      | Страница «Комната не найдена» + кнопка «Создать новую»      |
| Комната заполнена (2/2) | Страница «Комната заполнена. Подождите, пока кто-то выйдет» |
| Пустое имя              | Валидация: «Введите имя (минимум 2 символа)»                |
| Имя > 20 символов       | Обрезка до 20                                               |

## 11. СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ

---

### 11.1 Основной: «Вечер кино»

```
1.  Алексей → syncroom.onrender.com
2.  Вводит «Алексей» → «Создать комнату»
3.  POST /api/rooms → создана комната "x7k9p2m4"
4.  Редирект на /room/x7k9p2m4
5.  WebSocket connect → room:join { roomId, userName: "Алексей" }
6.  Копирует ссылку → отправляет Дмитрию

7.  Дмитрий открывает ссылку → /room/x7k9p2m4
8.  Вводит «Дмитрий»
9.  WebSocket connect → room:join { roomId, userName: "Дмитрий" }
10. Сервер → room:state (пустая комната) → room:userJoined { "Дмитрий" }
11. У Алексея toast: «Дмитрий присоединился» + системное сообщение в чате

12. Алексей вставляет ссылку на YouTube → [▶ Play]
13. video:change → оба видят видео → video:play
14. Оба смотрят синхронно

15. Дмитрий пишет «Класс! 😂» → chat:message → оба видят
16. Алексей ставит реакцию ❤️ → chat:reaction → оба видят

17. Алексей добавляет ещё 3 видео в плейлист
18. Первое видео заканчивается → video:ended → автоматически следующее

19. Через 3 часа оба закрывают вкладки
20. Всё сохранено в Neon: чат, плейлист, история
```

### 11.2 «На следующий день»

```
1. Алексей открывает ту же ссылку /room/x7k9p2m4
2. Имя автозаполнено из localStorage
3. room:join → сервер загружает из Neon:
   - последние 100 сообщений (вчерашний чат!)
   - плейлист (оставшиеся видео!)
   - состояние видео
4. Алексей видит весь вчерашний чат + плейлист
5. Кидает ссылку Дмитрию → продолжают
```

### 11.3 «Сервер перезапустился»

```
1. Render перезапустил сервер (деплой или maintenance)
2. RAM очищена — все комнаты потеряны из памяти
3. Клиенты видят «Переподключение...»
4. Socket.io автореконнект через 1 сек
5. room:join → сервер не находит комнату в RAM
6. Сервер загружает из Neon: комната + чат + плейлист
7. Кеширует в RAM
8. Отправляет клиентам полное состояние
9. Всё восстановлено, пользователи ничего не заметили
```
